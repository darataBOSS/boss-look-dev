# boss-look-dev 使い方ガイド

ライティングを土台に AR / VR / MR のリアルなルックを作るツール。メニュー **`BOSS > Look Dev`** でウィンドウを開く。

---

## 0. インストール

Package Manager → `+` → **Add package from git URL**:

```
https://github.com/darataBOSS/boss-look-dev.git
```

バージョン固定する場合はタグを付ける: `...boss-look-dev.git#v0.6.0`

- **URP / Post Processing Stack v2 はハード依存にしていません**。プロジェクトのパイプラインに合わせて自動で対応アダプタが立ち上がります（未インストールでもコンパイルは通る）。
- STYLY 向けは **Built-in プロジェクト**で制作（STYLY は URP/HDRP 非対応）。

---

## 1. 基本の流れ（最短）

1. `BOSS > Look Dev` を開く
2. **「＋ 新規 Look を作成」** → 保存先を選ぶ（= ルック設定一式を保存する `LookDefinition` アセットが1つできる）
3. 上部で **Target**（SelfApp / STYLY）と **Context**（VR / AR / MR）を選ぶ
4. **HDRI** を「ライティング」カードの HDRI スロットに入れる
5. **⚡ クイックスタート** … スカイボックス/IBL → ライト → プローブ → カラーまで一括
6. ライティング / カラー / フォグ / 背景を微調整
7. **🔥 ベイク**
8. **事故チェック（検証）** で実機の落とし穴を確認

> Look Definition = 「そのシーンのルックの保存ファイル」。ウィンドウはそれを編集するエディタ。1シーン1個が目安（複数作ってブレンド・切替も可）。

---

## 2. 各カードの役割

### ⚡ クイックスタート（良い既定値）
HDRI を入れて1ボタンで土台一式を構築。HDRI 未設定なら既存シーンのスカイボックスを使用。**ベイクプリセット**（📱モバイル / ✨高品質）も。

### ライティング（土台）
リアルさの土台。**HDRI 環境光**（強度・露出・回転はライブ反映）／**ベイク設定**（解像度・サンプル・バウンス・AO・バックエンド）／**プローブ**（範囲はシーンから自動も可）・**リフレクション**／**マテリアル底上げ**（白アルベド抑制・鏡面緩和・instancing）／**ライトリグ**（3点 / 太陽 / 天井グリッド、太陽は HDRI から向き推定可）／**🔥 ベイク**。

### 背景（カメラ）
ベイク後の **背景の見せ方**だけを切替（ライティングは保持）。
- **VR** = スカイボックス表示（HDRI が CG 背景）
- **AR** = 透過（実世界 / passthrough が背景）

### カラー
ライティングの上に乗せる色味。露出・コントラスト・彩度・カラーフィルター・色温度・Bloom・Vignette。**詳細設定**（折りたたみ）で threshold/scatter/tint・Hue・Tonemapping・LUT 等。Built-in=PPv2 / URP=Volume を自動切替。**STYLY ではポストを STYLY 側に委譲し、色はライティングに焼き込み**。

### フォグ（空気感・奥行き）
距離フォグ。VR 向け、AR/MR は通常オフ。**深海・水中の濁り表現の主役**になる。

### AR グラウンドシャドウ（AR 時のみ）
影だけを受ける透明な床。AR で「浮いて見える」のを防ぐ。

### VR↔MR 切り替え（State）
体験中に VR↔MR を切替える仕組み（→ §3）。

### ルックの比較・ブレンド
2つの Look を混ぜる（A→B）／今の状態を保存して変更前後を見比べる（before/after スナップショット）。

### 事故チェック（検証）
実機で事故りやすい設定を警告：未ベイク / 純白アルベド / UV2 欠落 / 色空間 / カメラHDR / 色温度 / プローブ欠落 / STYLY 制約 など。納品前に自分で気付くため。

---

## 3. VR↔MR 切り替え（State トグル方式）

体験中に VR↔MR を切り替えたいとき。**ベイクは2状態で共有**し、差分（背景・フォグ・環境光・反射）だけを切替える。

1. 「VR↔MR 切り替え (State)」で **State 切り替えを使う** を ON、状態A/B の差分を設定
2. **State リグを生成** → `〇〇 Switch` の下に `VR_State` / `MR_State`（空オブジェクト）が生成され、**常に片方だけアクティブ**
3. **📄 ハンドオフ手順書を生成** → エンジニアに渡す日本語手順書（`〇〇_Handoff.md`）

**切り替え方**（手順書にも記載）:
- **SelfApp**: `VR_State` / `MR_State` を `SetActive` で切替（`BossLookDevState` が適用＋スムーズ transition）。Timeline でも可。
- **STYLY**: PlayMaker の **Activate Game Object** で切替（瞬時）。フォグ等のグローバルは PlayMaker アクションで。

> 没入モード（passthrough）自体の VR↔MR 切替は Pico/Quest SDK 側でエンジニアが制御。本ツールは**コンテンツのルック**を切り替える。同じタイミングで合わせる。

---

## 4. ターゲット別メモ

| | SelfApp（Pico/Quest 自前アプリ） | STYLY（PlayMaker） |
|---|---|---|
| パイプライン | URP（推奨）/ Built-in | **Built-in のみ** |
| カラー | PPv2 / URP Volume をフル使用 | ポストは STYLY 側へ委譲、色はライティング焼き込み |
| 実行時 | C# 可（State コンポーネント・transition） | カスタム C# 不可（PlayMaker のみ・瞬時切替） |
| レイヤー | 任意 | Default を使用 |

---

## 5. ヒント

- リアルさは **全ベイク＋質感の作り込み**で出す（HDRP 的リアルタイムは不可）。HDRI-IBL → ベイク → reflection probe が土台。
- **MSAA を優先**（post AA より）。DoF・Motion Blur・SSAO はモバイル XR 方針で非対応。
- 純白アルベドは GI で白飛びの原因 → マテリアル底上げ or 検証で 0.85 に。
- 最終チェックは**実機**（Pico / Quest）で。モニタはステレオ・色域・スケールで嘘をつく。
